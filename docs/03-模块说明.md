# 模块说明

## 模块1：Domain 层 (`app/domain/`)

### 概述
定义项目的核心领域模型、枚举和评分逻辑。所有模型基于 Pydantic v2，贯穿 API、Service、Agent 各层。

### 主要文件

| 文件 | 说明 |
|------|------|
| `models.py` | 核心领域模型定义 |
| `schemas.py` | API 请求/响应 Schema |
| `enums.py` | 枚举定义（角色类型、冲突类型、方案类型等） |
| `scoring.py` | 方案评分与推荐逻辑 |

### 核心模型

#### PlanOutput（聚合根）
```python
class PlanOutput(DomainModel):
    session_id: str               # 会话 ID
    user_id: str                  # 用户 ID
    input_query: str              # 原始用户输入
    inferred_context: GroupContext # 推断的群体上下文
    conflicts: list[Conflict]     # 检测到的冲突
    negotiation_strategies: list[NegotiationStrategy]  # 协商策略
    plan_candidates: list[PlanCandidate]  # 候选方案
    recommended_plan_id: str      # 推荐方案 ID
    execution_graph: list[ExecutionTask]  # 执行任务图
    plan_version: int             # 方案版本（重规划时递增）
    state: PlanState              # 方案状态
```

#### GroupContext（群体上下文）
```python
class GroupContext(DomainModel):
    group_type: GroupType         # 群体类型（家庭/朋友/情侣等）
    roles: list[RoleProfile]      # 角色列表
    group_size: int               # 群体大小
    scene_label: str              # 场景标签
    inferred_constraints: list[str]  # 推断的约束
```

#### RoleProfile（角色画像）
```python
class RoleProfile(DomainModel):
    role_type: RoleType           # 角色类型（儿童/成人/老人等）
    hard_constraints: list[str]   # 硬约束（必须满足）
    soft_preferences: list[str]   # 软偏好（尽量满足）
    hidden_needs: list[str]       # 隐性需求
    priority_weight: float        # 优先级权重
```

#### POI（兴趣点）
```python
class POI(DomainModel):
    id: str
    name: str
    category: str                 # 分类
    city: str
    area: str | None              # 区域
    avg_price: int | None         # 均价
    open_hours: str | None        # 营业时间
    energy_level: int             # 能量等级 0-5
    crowd_risk: str               # 拥挤风险
    queue_risk: str               # 排队风险
    suitable_for: list[str]       # 适合人群
    experience_scores: ExperienceScores  # 体验评分（出片/聊天/新奇/放松）
```

#### PlanCandidate（候选方案）
```python
class PlanCandidate(DomainModel):
    plan_type: PlanType           # Plan A / Plan B / Recommended
    title: str
    theme: str
    stages: list[Stage]           # 阶段列表
    timeline: list[TimelineItem]  # 时间轴
    satisfaction_scores: list[SatisfactionScore]  # 各角色满意度
    overall_score: float          # 综合评分
    min_role_score: float         # 最低角色评分
    fairness_score: float         # 公平性评分
```

## 模块2：Agent 编排层 (`app/agent/`)

### 概述
核心 AI 编排层，通过 Orchestrator 驱动 8 个 Skill 串行执行，完成从用户理解到方案输出的完整规划流程。

### 主要文件

| 文件 | 说明 |
|------|------|
| `orchestrator.py` | 核心编排器，管理 8 步流水线 |
| `plan_manager.py` | 计划生命周期管理（创建/更新/确认） |
| `session_store.py` | 会话存储（内存字典） |

### Skills（`agent/skills/`）

| Skill | 说明 |
|-------|------|
| `user_understanding.py` | 解析用户意图，生成群体上下文 |
| `conflict_detection.py` | 识别角色间利益冲突 |
| `negotiation.py` | 生成协商策略（补偿/阶段策略） |
| `experience_planning.py` | 生成体验骨架和候选方案 |
| `place_selection.py` | 为候选方案选择主备 POI |
| `routing.py` | 规划 POI 间转场路线 |
| `timeline_builder.py` | 构建详细时间轴 |
| `scoring` (domain) | 评分推荐（非 Skill，属于 Domain 层） |

### 编排流水线

```
用户输入
  │
  ▼
① UserUnderstandingSkill ──▶ GroupContext（群体画像）
  │
  ▼
② ConflictDetectionSkill ──▶ Conflict[]（冲突列表）
  │
  ▼
③ NegotiationSkill ──▶ NegotiationStrategy[]（协商策略）
  │
  ▼
④ ExperiencePlanningSkill ──▶ PlanCandidate[]（候选方案骨架）
  │
  ▼
⑤ PlaceSelectionSkill ──▶ 为每个 Stage 选择主备 POI
  │
  ▼
⑥ RoutingSkill ──▶ 规划转场路线
  │
  ▼
⑦ TimelineBuilderSkill ──▶ 构建时间轴
  │
  ▼
⑧ Scoring ──▶ 评分 + 选出推荐方案
  │
  ▼
构建 ExecutionTask 图 ──▶ PlanOutput
```

## 模块5：API 层 (`app/api/`)

### 概述
FastAPI 路由层，负责 HTTP 请求路由和参数校验。通过依赖注入获取 Service 实例。

### 主要文件

| 文件 | 说明 |
|------|------|
| `deps.py` | 依赖注入工厂（根据配置创建 Service 实例） |
| `v1/router.py` | 路由聚合 + 健康检查 + 元信息端点 |
| `v1/plans.py` | 规划端点（预览、流式预览、查询、确认） |
| `v1/events.py` | 事件触发端点 |
| `v1/execution.py` | 执行端点 |
| `v1/feedback.py` | 反馈端点 |

---

### 概述
