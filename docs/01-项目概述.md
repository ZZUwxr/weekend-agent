# Weekend Agent - 项目概述

## 项目简介

Weekend Agent 是一个**本地探索 / 周末活动规划 Agent 后端系统**，专注于群体协商型活动规划。系统从用户的自然语言输入中识别同行角色、硬约束、软偏好和内部冲突，通过 LLM 驱动的多技能编排流水线，自动生成 Plan A / Plan B / 推荐方案，并提供满意度评分、时间轴和可执行任务图。

## 核心功能

- **自然语言理解**：从用户描述中提取群体结构（角色、年龄、约束、偏好）
- **冲突检测与协商**：识别群体成员间的利益冲突（如减肥 vs 美食、儿童安全 vs 冒险体验），生成协商策略
- **智能行程规划**：自动生成多个候选方案，包含 POI 选择、转场路线、时间轴
- **满意度评分**：为每个角色评估方案满意度，计算公平性评分，选出推荐方案
- **SSE 流式输出**：支持 Server-Sent Events 实时推送规划进度
- **事件驱动重规划**：支持排队变更、天气变化等事件触发方案重规划
- **执行任务图**：生成预约、叫车、分享等可执行任务，支持 Mock 执行
- **用户反馈收集**：支持评分和文本反馈

## 快速开始

### 环境要求

- Python >= 3.11
- conda 或 venv 虚拟环境
- （可选）PostgreSQL + PostGIS（使用 json 数据后端时不需要）

### 安装与运行

```bash
# 克隆项目
git clone <repo-url>
cd weekend-agent

# 创建并激活虚拟环境
conda create -n weekend-agent python=3.11
conda activate weekend-agent

# 安装依赖
pip install -e .

# 配置环境变量（可选，默认 Mock 模式无需配置）
cp .env.example .env

# 启动服务
python -m uvicorn local_explorer_agent.app.main:app --reload
```

### 验证服务

```bash
# 健康检查
curl http://127.0.0.1:8000/api/v1/health

# 数据健康检查
curl http://127.0.0.1:8000/api/v1/meta/data-health

# 生成规划示例
curl -X POST http://127.0.0.1:8000/api/v1/plans/preview \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "u001",
    "query": "今天下午想和老婆孩子出去玩几小时，别太远，老婆最近在减肥，孩子5岁",
    "city": "深圳",
    "start_time": "2026-05-10T14:00:00",
    "duration_minutes": 240,
    "location": {"lat": 22.54, "lon": 114.05}
  }'
```
