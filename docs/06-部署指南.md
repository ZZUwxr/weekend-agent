# 部署指南

## 环境要求

| 项目 | 要求 |
|------|------|
| Python | >= 3.11 |
| 操作系统 | Linux / macOS / Windows (WSL) |
| 内存 | >= 2GB |
| 磁盘 | >= 500MB |
| （可选）PostgreSQL | >= 14 + PostGIS 扩展 |

## 环境变量配置

创建 `.env` 文件（可从 `.env.example` 复制）：

```env
# 应用基础配置
APP_NAME=Weekend Agent
APP_ENV=local

# LLM 配置
LLM_PROVIDER=openai              # mock（无需 API Key）或 openai
LLM_API_KEY=                     # LLM API Key，本地填写，不要提交真实值
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen3.5-plus           # 模型名称
LLM_TIMEOUT_SECONDS=60           # 超时时间
LLM_MAX_RETRIES=2                # 最大重试次数
LLM_TEMPERATURE=0.2              # 温度参数
LLM_USE_STRUCTURED_OUTPUT=true   # 使用结构化输出
LLM_API_STYLE=chat_completions   # API 风格
LLM_TRUST_ENV=false              # 是否信任代理环境变量
LLM_ALLOW_RULE_BASED_FALLBACK=false  # 是否允许规则引擎降级

# 数据配置
DATA_DIR=local_explorer_agent/app/data   # 数据目录
DATA_BACKEND=json                        # 数据后端：json 或 postgres

# 数据库配置（仅 DATA_BACKEND=postgres 时需要）
DATABASE_URL=postgresql://user:password@localhost:5433/weekend_agent
```

### LLM 配置说明

**Mock 模式（开发/测试）**
```env
LLM_PROVIDER=mock
```
无需 API Key，使用规则引擎生成结构化输出，适合开发和端到端测试。

**OpenAI 官方 API**
```env
LLM_PROVIDER=openai
LLM_API_KEY=
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
LLM_API_STYLE=chat_completions
LLM_TRUST_ENV=false
```

**阿里云 DashScope（推荐）**
```env
LLM_PROVIDER=openai
LLM_API_KEY=
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen3.5-plus
LLM_API_STYLE=chat_completions
LLM_TRUST_ENV=false
```

**其他 OpenAI 兼容服务（vLLM / OneAPI / LiteLLM）**
```env
LLM_PROVIDER=openai
LLM_API_KEY=
LLM_BASE_URL=http://localhost:8000/v1
LLM_MODEL=your-model-name
LLM_API_STYLE=chat_completions
```

## 构建与运行

### 开发环境

```bash
# 1. 创建虚拟环境
conda create -n weekend-agent python=3.11
conda activate weekend-agent

# 2. 安装依赖（开发模式）
pip install -e .

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 LLM API Key（可选，mock 模式无需）

# 4. 启动开发服务器
python -m uvicorn local_explorer_agent.app.main:app --reload

# 5. 验证
curl http://127.0.0.1:8000/api/v1/health
```

### 生产环境

```bash
# 1. 安装依赖
pip install -e .

# 2. 配置环境变量
export LLM_PROVIDER=openai
export LLM_API_KEY=
export LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
export LLM_MODEL=qwen3.5-plus
export DATA_BACKEND=json

# 3. 启动服务（生产模式）
python -m uvicorn local_explorer_agent.app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 数据后端切换

**JSON 后端（默认，零依赖）**
```env
DATA_BACKEND=json
DATA_DIR=local_explorer_agent/app/data
```

Repository 会优先读取完整数据文件（如 `poi.json`），不存在时回退 `.sample.json`。

**PostgreSQL 后端**
```env
DATA_BACKEND=postgres
DATABASE_URL=postgresql://user:password@localhost:5433/weekend_agent
```

需要先初始化数据库：
```bash
# 创建数据库
createdb weekend_agent

# 启用 PostGIS 扩展
psql -d weekend_agent -c "CREATE EXTENSION IF NOT EXISTS postgis;"

# 导入表结构
psql -d weekend_agent < database/schema.sql

# 导入数据
python database/import_generated_data.py
```

## 数据健康检查

```bash
# 检查数据文件状态
curl http://127.0.0.1:8000/api/v1/meta/data-health
```

返回每类数据文件是否存在、记录数量、缺失关键字段数量。少量字段缺失不会阻止服务运行。

## 常见问题

### 问题1：LLM API Key 未配置导致规划失败

**现象**：调用 `/plans/preview` 返回错误

**原因**：`LLM_PROVIDER=openai` 但未配置 `LLM_API_KEY`

**解决方案**：
- 方案A：配置有效的 `LLM_API_KEY`
- 方案B：切换为 Mock 模式 `LLM_PROVIDER=mock`
- 方案C：设置 `LLM_ALLOW_RULE_BASED_FALLBACK=true` 允许降级

### 问题2：数据文件不存在

**现象**：`/meta/data-health` 显示文件缺失

**原因**：`DATA_DIR` 路径不正确或数据文件未生成

**解决方案**：
- 确认 `DATA_DIR` 路径正确
- `.sample.json` 文件已包含在仓库中，会自动回退使用
- 如需完整数据，运行 `data_process/generated_data_byllm.py` 生成

### 问题3：PostGIS 扩展未安装

**现象**：导入 schema.sql 报错

**原因**：PostgreSQL 未安装 PostGIS 扩展

**解决方案**：
```bash
# Ubuntu/Debian
sudo apt install postgis postgresql-14-postgis-3

# macOS
brew install postgis

# 然后启用扩展
psql -d weekend_agent -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

### 问题4：SSE 流式请求超时

**现象**：流式预览请求中途断开

**原因**：反向代理（如 Nginx）缓冲了 SSE 响应

**解决方案**：
- Nginx 配置：`proxy_buffering off;`
- 确保响应头包含 `Cache-Control: no-cache` 和 `X-Accel-Buffering: no`
